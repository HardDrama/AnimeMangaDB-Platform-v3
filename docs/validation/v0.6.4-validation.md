## Scope

Version 0.6.4 continues the installment query-service foundation by
validating the service layer against the repository interfaces used by
episode and chapter installments.

The completed work verifies that:

- Episode query services use the episode repository contract.
- Chapter query services use the chapter repository contract.
- Installments can be retrieved by identifier.
- Installments can be listed for their owning title.
- Missing installment results are handled consistently.
- Repository protocol tests reflect the concrete repository APIs.
- Service behavior remains isolated from SQLAlchemy implementation details.

## Repository Contracts

### EpisodeRepository

The episode repository exposes:

- `add`
- `get_by_identifier`
- `exists_by_identifier`
- `list_for_anime_title`
- `delete`
- `_resolve_anime_title_orm`

### ChapterRepository

The chapter repository exposes:

- `add`
- `get_by_identifier`
- `exists_by_identifier`
- `list_for_manga_title`
- `delete`
- `_resolve_manga_title_orm`

The query-service protocols intentionally use:

- `EpisodeRepository.list_for_anime_title`
- `ChapterRepository.list_for_manga_title`

A generic `list_for_title` repository method is not required.
