## Implementation Notes

- Repositories stage work but do not implicitly commit.
- Transaction completion remains the responsibility of the caller.
- Explicit `flush()` calls are used only where dependent repositories must resolve parent
  rows inside the same transaction.
- Repository reconstruction creates new domain objects and preserves observable domain
  state.
- Domain equality semantics are intentionally not dictated by database certification.
- ORM primary keys and SQLAlchemy models remain infrastructure concerns.
